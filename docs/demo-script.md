# Case Compass — Demo Script

**Audience**: Hackathon judges. Walked at 14:45 rehearsal and 15:30 judging.
**Duration**: ~2.5 minutes spoken, ~6 clicks.
**Pre-flight**: `bun dev` is running on `http://localhost:8080`. Browser is open at `/`. Cookies/localStorage cleared so the user looks like a first-time judge. `VITE_SUPABASE_URL` and `VITE_SUPABASE_ANON_KEY` are set in `.env.local` (the OpenAI key lives server-side on the `priority-insight` Supabase Edge Function — never in the browser bundle), and the OpenAI account has billing enabled. The assistant sentence and the GOV.UK guidance panel on the case detail page are the only live external calls.

---

## Beat 1 — The hook (15 seconds)

> "Every UK government caseworker loses about five seconds reconstructing context every time they open a case. Multiply that across a caseload, a team, a department — that's millions of decisions, billions of seconds, lost to tab-switching. Case Compass closes the gap on a single screen."

Open `/`. Pause for the table to render.

## Beat 2 — The morning briefing (20 seconds)

Point at the AI Strategy Assistant sidebar on the right.

> "Sam, the caseworker, opens the dashboard. She doesn't read a single row. The AI Strategy Assistant has already done the morning triage — Red, Amber, Green counts at the top, and one priority insight: 'CASE-2026-00042 has breached POL-INC-01 — recommended action: Issue reminder.' She knows where to start in under five seconds."

Pause for the sentence to render (the LLM call should resolve within 1-2 seconds).

> "The selection of which case is the priority is fully deterministic — same fixture, same answer, every render. Only the human-readable sentence is generated by the LLM at render time. If the API key is missing or the call fails, the panel falls back to the same sentence in deterministic form. The demo path never breaks."

## Beat 3 — Heatmap and table (15 seconds)

Point at the workload heatmap above the caseload table.

> "For Priya, the team leader, the heatmap above the table answers a different question: how bad is today, in one glance? Reds first, then amber, then green. She can see the shape of the day without scrolling."

Point at the four summary tiles, then the table.

> "Below the heatmap: total cases, awaiting evidence, overdue, average age — all reflecting the current filter. Then the caseload table, sorted risk-descending. CASE-2026-00042 — Jordan Smith — top of the list, red badge, 10 out of 10. Hover the badge: past escalation threshold, five overdue evidence items, recency since last update."

## Beat 4 — Drill in (25 seconds)

Click the assistant's primary CTA (or the matching heatmap tile, or the top table row — all three lead to the same place).

> "She lands on Jordan's case detail. One screen. Top-left: applicant, reference, the case is 96 days old. Top-right: the same risk badge, with the same factor breakdown."

Scan down the timeline column.

> "Timeline on the left — the case was opened on 2026-01-10, evidence requested 2026-01-15, nothing since."

Move to the right column.

> "Workflow state and required next actions on the right. The system surfaces 'Issue reminder' as the critical action, ordered above the warnings — derived from the workflow state machine *and* the overdue evidence count, not hardcoded."

Hover the Evidence Tracker.

> "Five rows in the evidence tracker. Every one says outstanding or overdue. Each shows the source policy ID and the days elapsed against the threshold — 91 days against a 56-day threshold for the income statement."

Scroll to Applicable Policy.

> "And the policy panel below shows the actual policy text the requirements were parsed from. The cross-reference is live, not pre-baked."

## Beat 5 — Take action (15 seconds)

Click the "Issue reminder" action button.

> "Click any next action — land on a stub page with the specific GOV.UK pages a caseworker would need to navigate to actually issue that reminder. Read-only — no state mutation, this is a hackathon prototype, not a write-enabled tool."

Click the back link.

## Beat 6 — Round trip (15 seconds)

Back on case detail. Click the "All cases" breadcrumb in the case header (or the header wordmark).

> "Back to the dashboard. Same data, same sort order, same risk scores — every derivation is a pure function of the fixture inputs against a frozen reference date. The numbers you saw at 14:45 rehearsal will be the numbers you see now. The assistant's priority case is unchanged — same selection rule, same fixture inputs. That's why we trust the system enough to act on it."

## Pitch close (15 seconds)

> "One screen, every signal derived from authoritative inputs, no tab-switching, no reconstruction work. The five seconds × millions framing isn't a slogan — it's the calculation that justifies the build."

## Failure recovery

| Symptom | Recovery |
|--------|----------|
| Top row is not `CASE-2026-00042` | Click the "Risk" header once — re-sorts risk-descending |
| GOV.UK guidance panel shows error | Acknowledge it as a live API call; it does not block the demo path |
| Browser console shows red errors | Note the case ID + page; complete demo, fix after |
| Action stub link 404s | The action ID may have changed; click any other action — the stub is route-driven, not action-specific |
| AI Strategy Assistant shows the deterministic fallback sentence | Note "the edge-function call failed; the deterministic fallback is what FR-119c protects against — the demo path still works." Do not panic-debug |
| AI Strategy Assistant shows "LLM assistant not configured" | Check `VITE_SUPABASE_URL` and `VITE_SUPABASE_ANON_KEY` in `.env.local` and that `bun dev` was restarted after the values were added; verify the `priority-insight` edge function is deployed and that `OPENAI_API_KEY` is set as a Supabase secret. In a pinch, frame the panel as the deterministic-only mode |
| Heatmap tile colours look off | The risk levels feed both the table badge and the heatmap from the same source; if one is wrong, both are wrong — flag as a derivation bug |

## Hard rules from the constitution

- No code changes between rehearsal (14:45) and judging (15:30). SC-005.
- No live writes. Anything that looks like persistence is a bug to flag.
- The reference date is `2026-04-16`. Don't override it during the demo.
